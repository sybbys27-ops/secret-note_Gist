### 🔐 1. 핵심 보안 구조 (밀리터리급 암호화)

이 앱의 가장 큰 특징은 **사용자의 브라우저 저장소(`localStorage`)에 원본 글(평문)을 절대 남기지 않는다**는 점입니다.

* **암호화 알고리즘 (AES-GCM 256-bit):** 현재 금융권과 보안 업계에서 표준으로 사용하는 가장 안전하고 빠른 암호화 방식을 사용했습니다.
* **키 생성 방식 (PBKDF2):** 사용자가 입력한 비밀번호를 그대로 쓰지 않고, 해시 함수를 150,000번 반복(Iterations)하여 해킹(무차별 대입 공격)을 방지하는 강력한 마스터 암호키로 변환합니다.
* **영지식(Zero-Knowledge) 증명 구조:** 마스터 비밀번호를 분실하면 제작자 본인조차도 절대 데이터를 복구할 수 없는 완벽한 철통 보안 구조를 가지고 있습니다.

### 📝 2. 주요 기능 및 UI 분석

다크 모드 기반의 세련된 UI(`--bg: #0f1115`)를 채택하여 해커의 터미널 같은 멋진 분위기를 풍깁니다.

* **잠금/해제 시스템:** 처음 접속 시 'LOCKED' 상태이며, 비밀번호를 입력해 성공적으로 복호화(해독)해야만 전체 UI가 활성화됩니다.
* **목록 관리:** 즐겨찾기(⭐ 핀 고정) 기능과 정렬(최근 수정순, 이름순) 기능을 제공합니다.
* **단축키 지원:** 개발자나 파워 유저를 위해 잠금 해제 상태에서 내용 작성 후 `Ctrl + S`를 누르면 즉시 암호화되어 저장되도록 설계되었습니다.
* **로컬 백업/복원:** [내보내기]를 하면 JSON 파일로 다운로드되는데, 이 파일 안의 데이터조차 **모두 암호문(Base64) 상태**이므로 USB에 담아 다녀도 매우 안전합니다.

### 💻 3. 코드 설계 요약

* **Crypto API 활용:** 자바스크립트의 내장 `crypto.subtle` API를 사용하여 외부 라이브러리 없이 순수 브라우저 기능만으로 고성능 암호화를 구현했습니다.
* **상태 관리 (State & Session):** `state` 객체에는 암호화된 파일만 담아서 로컬스토리지에 저장하고, 비밀번호와 해독된 데이터는 브라우저를 끄면 즉시 날아가는 휘발성 `session` 메모리에만 담아두어 보안을 극대화했습니다.

## 사용방법

드디어 나만의 **'클라우드 비밀 노트'** 파일이 완성되었군요! 🎉
이름도 `secret-note_Gist.html`로 아주 직관적이고 완벽하게 지으셨습니다.

이제 이 파일을 이용해 **첫 암호화 노트를 작성하고, 구름(Gist)에 안전하게 띄운 뒤, 다른 PC에서 불러오는 전체 흐름**을 차근차근 안내해 드릴게요. 천천히 따라와 보세요!

---

### 🚀 1단계: 첫 실행 및 마스터 비밀번호 설정

1. 방금 저장하신 `secret-note_Gist.html` 파일을 더블클릭해서 브라우저(크롬, 엣지 등)로 엽니다.
2. 상단 입력창에 앞으로 사용할 '마스터 비밀번호'를 입력하고 **[잠금 해제]** 버튼을 누릅니다.
* *⚠️ 주의: 이 비밀번호는 어딘가에 저장되지 않습니다. 잊어버리면 그 누구도(본인조차도) 노트를 복구할 수 없으니 꼭 기억할 수 있는 비밀번호로 설정하세요!*



### 📝 2단계: 테스트 비밀 일기 작성하기

1. 왼쪽 목록 창 위에 있는 **[+ 새 항목]** 버튼을 클릭합니다.
2. 오른쪽 화면에 제목(예: `비밀/첫 동기화 테스트`)과 내용(`구름 동기화가 잘 되는지 테스트 중입니다.`)을 자유롭게 적습니다.
3. **[저장]** 버튼을 누릅니다. (이제 내 PC의 로컬 금고에 '암호화된 상태'로 안전하게 저장되었습니다.)

### ☁️ 3단계: 구름(Gist) 동기화 세팅하기

1. 화면 중앙 쯤에 있는 **[⚙️] (설정)** 버튼을 누릅니다.
2. 첫 번째 칸에 깃허브에서 발급받은 GitHub Token (`ghp_...`)을 붙여넣습니다. (기존 앱들에서 쓰시던 그 토큰 맞습니다!)
3. **🚨 중요:** 두 번째 칸인 **'Gist ID (방 번호)'는 반드시 비워둡니다.** (아직 이 비밀 노트를 위한 전용 방을 안 만들었으니까요!)
4. **[저장]** 버튼을 눌러 설정창을 닫습니다.

### 📤 4단계: 구름으로 첫 업로드 (방 만들기)

1. **[구름저장☁️]** 버튼을 클릭합니다.
2. "저장중..." 메시지가 뜨고 잠시 후, **"새로운 구름 저장소가 생성되었습니다! Gist ID: b3b3..."** 라는 알림창이 뜹니다.
3. 성공입니다! 이제 이 비밀 노트만의 고유한 Gist ID(방 번호)가 발급되었고, 내 PC의 설정창에도 방 번호가 자동으로 입력되었습니다.

---

### 💡 (참고) 다른 PC나 폰에서 불러올 때는 어떻게 하나요?

회사 PC나 스마트폰에서 이 비밀 노트를 열고 싶다면 아래 순서대로 하시면 됩니다.

1. `secret-note_Gist.html` 파일을 엽니다. (또는 깃허브 Pages에 올려둔 주소로 접속합니다.)
2. 먼저 [⚙️] (설정)을 눌러서 **토큰**과 아까 발급받은 Gist ID(방 번호)를 둘 다 입력하고 저장합니다.
3. [불러오기☁️]를 누릅니다. ("구름에서 데이터 불러오기 완료" 메시지가 뜹니다.)
4. 하지만 목록은 여전히 비어있을 겁니다! (방어 시스템 작동 중 🛡️)
5. 상단에 '마스터 비밀번호'를 입력하고 [잠금 해제]를 누릅니다.
6. 짠! 🔒 잠금이 풀리면서 구름에서 내려온 비밀 노트 목록이 완벽하게 나타납니다.
